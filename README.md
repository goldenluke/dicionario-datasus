# 📊 Dicionário de Variáveis do DATASUS

> Um guia prático, crítico e em construção para compreender melhor os dados públicos de saúde no Brasil.

---

## 🧠 Sobre o projeto

O **Dicionário de Variáveis do DATASUS** nasceu a partir de uma inquietação simples:

> Os dados estão disponíveis. Mas será que estamos realmente entendendo o que eles significam?

Ao trabalhar com diferentes bases do DATASUS, ficou evidente que o maior desafio não é acessar os dados —  
é **interpretá-los com rigor, contexto e responsabilidade**.

Este repositório é uma tentativa de organizar esse entendimento.

Não se trata apenas de documentar variáveis, mas de explorar:

- o **significado real** dos campos  
- as **limitações dos dados**  
- as **armadilhas analíticas mais comuns**  
- e o **contexto epidemiológico e operacional** por trás das bases  

---

## 🗂️ Bases contempladas

O projeto abrange múltiplos sistemas do DATASUS.

Cada base é tratada como um sistema próprio, com:

- lógica interna  
- estrutura específica  
- limitações particulares

## 🔍 O que você vai encontrar

### 📌 Dicionário de variáveis
- Descrição detalhada dos campos
- Tipos de dados e codificações
- Interpretação prática

### ⚠️ Notas críticas
- Problemas de completude
- Campos com alta taxa de “ignorado”
- Variáveis inconsistentes ao longo do tempo

### 🔄 Mudanças históricas
- Alterações estruturais nas bases
- Quebras de série temporal
- Diferenças entre versões

### 🧪 Guia de análise
- Boas práticas no uso dos dados
- Cuidados metodológicos
- Exemplos de erros comuns

### 🔗 Possibilidades de integração
- Linkage entre bases (ex: SINASC + SIM)
- Uso conjunto com CNES
- Expansão analítica

## 🎯 Objetivo

Este projeto não busca ser definitivo.

Ele busca ser **útil**.

A ideia é oferecer um material que ajude a:

- reduzir erros de interpretação  
- aumentar o rigor analítico  
- apoiar estudos em saúde pública  
- incentivar um uso mais crítico dos dados do SUS  

---

## 🧩 Filosofia do projeto

- Dados não são neutros  
- Variáveis não são autoexplicativas  
- “Ignorado” também é informação  
- Mudanças de formulário importam  
- Contexto é tão importante quanto código
